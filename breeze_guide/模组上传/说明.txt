这里是微风指南的开发者模组上传目录。

玩家端没有上传入口。只有拥有仓库写入权限的维护者，才能把模组压缩包放入此目录并修改上一级的“模组发布清单.json”。

推荐流程：
1. 把完整模组 ZIP 放到本目录，不要先改动模组原有目录结构。
2. 在“模组发布清单.json”的“上传模组”数组中加入一项，例如：
   {
     "文件": "穆尔卡的武器工坊.zip",
     "发布": true,
     "默认启用": false
   }
3. 提交到功能分支，检查 GitHub Actions 的“构建并发布微风指南”。
4. 验证通过后再合并到 test/next-release。

可选字段：
“模组标识”：一个压缩包含多个模组时，只发布指定 MOD_INFO id。
“名称”：覆盖网页中显示的模组名称。
“说明”：覆盖网页中显示的模组说明。
“替换同名内置模组”：设为 true 时，允许上传版本替换 data/mods 中相同 id 的版本。

安全规则：
只有清单中“发布”为 true 的文件会进入索引。
构建脚本会拒绝带有绝对路径或“..”越界路径的 ZIP。
网页只读取构建后的 JSON 数据，不会运行模组中的脚本或可执行文件。
